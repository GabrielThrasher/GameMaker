Glossary


Below are the 20 original basketball statistics gathered from our APIs. If in a per-player database table, each statistic corresponds to that person's alone. If in a per-team database table, each statistic corresponds to the average of all per-player statistics except MIN which corresponds to the sum of the per-player values.


  • MIN: Minutes — Minutes played on the court
  • FGM: Field Goals Made
  • FGA: Field Goals Attempted
  • FG_PCT: Field Goal Percentage — FGM divided by FGA
  • FG3M: 3-Point Field Goals Made
  • FG3A: 3-Point Field Goals Attempted
  • FG3_PCT: 3-Point Percentage — FG3M divided by FG3A
  • FTM: Free Throws Made
  • FTA: Free Throws Attempted
  • FT_PCT: Free Throw Percentage — FTM divided by FTA
  • OREB: Offensive Rebounds — Rebounds grabbed off your team’s missed shots
  • DREB: Defensive Rebounds — Rebounds grabbed after opponent misses
  • REB: Total Rebounds — Sum of OREB and DREB
  • AST: Assists — Passes leading directly to a made basket
  • STL: Steals — How many times this player took the ball from an opponent
  • BLK: Blocks — Shots blocked while defending
  • TO: Turnovers — Times the player lost the ball to the other team
  • PF: Personal Fouls — Number of fouls committed by the player
  • PTS: Total points scored
  • PLUS-MINUS: Net team score while this player was on the court

The following are common metrics used throughout the program, defined:


  • True Positives (TP): The number of instances where the model correctly predicted the positive class. Example: Model predicts a game will be a win, and it actually is a win.
  • True Negatives (TN): The number of instances where the model correctly predicted the negative class. Example: Model predicts a loss, and it indeed ends up being a loss.
  • False Positives (FP): The number of instances where the model incorrectly predicted the positive class (a "false alarm"). Example: Model predicts a win, but the game is actually a loss.
  • False Negative (FN): The number of instances where the model incorrectly predicted the negative class (a "missed detection"). Example: Model predicts a loss, but the game is actually a win.
  • Confusion Matrix (binary outcome case): A 2×2 table that summarizes the counts of TP, TN, FP, and FN. It provides a full breakdown of the model’s performance across all outcomes. A visual breakdown will be presented below.
  • Accuracy: The proportion of total predictions that were correct. Used when classes are roughly balanced. This metric is normalized to a range from 0 to 1, where a value of 1 indicates optimal performance. \( Accuracy = \frac{TP + TN}{TP + TN + FP + FN} \)
  • Recall: Measures how well the model identifies actual positives. Useful when missing positives is costly (e.g., predicting wins correctly). This metric is normalized to a range from 0 to 1, where a value of 1 indicates optimal performance. \( Recall = \frac{TP}{TP + FN} \)
  • Precision: Measures how many of the predicted positives are actually correct. Useful when false positives are costly (e.g., betting on predicted wins). This metric is normalized to a range from 0 to 1, where a value of 1 indicates optimal performance. \( Precision = \frac{TP}{TP + FP} \)
  • F1 Score: The harmonic mean of precision and recall. It balances both metrics into a single score. Best used when you need a trade-off between precision and recall. This metric is normalized to a range from 0 to 1, where a value of 1 indicates optimal performance. \( F1 = \frac{2*Recall*Precision}{Recall+Precision} \)
